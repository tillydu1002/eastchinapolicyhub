## 部署方式 · 二选一

### 方式 A：网页上传（0 命令行，推荐先跑一遍）

**前置**：GitHub 账号一个（没有就去 github.com 注册，免费）。

**步骤**：

1. 登录 GitHub，右上角 `+` → `New repository`
2. Repository name：随便取，比如 `ec-pa-briefing`
   - Visibility：**Public**（GitHub Pages 免费版必须公开。如果要私有可以走 Private + 付费 Pro，或后续迁到别处）
   - 不要勾 "Add a README"
   - 点 `Create repository`
3. 在刚创建的空仓库页面，点 `uploading an existing file`
4. 把 `site/` 目录下**所有内容**拖进去（注意是 site 里面的文件，不是 site 文件夹本身）
   - 包括：`index.html`、`archive/`、`data/`、`scripts/`、`.nojekyll`、`README.md`、`预览.bat`
   - `.nojekyll` 前缀有点特殊，如果拖不上去，上传完后点右上 `Add file → Create new file`，文件名填 `.nojekyll`，内容留空，提交
5. 页面底部 `Commit changes` → 提交
6. 仓库首页 → `Settings` → 左边 `Pages`
   - Source: `Deploy from a branch`
   - Branch: `main` / Folder: `/ (root)`
   - `Save`
7. 等 1-2 分钟，刷新 Pages 设置页，顶部会出现：
   > Your site is live at **https://你的用户名.github.io/ec-pa-briefing/**

这个链接就是**永久稳定链接**，发给同事就行。

### 方式 B：命令行（要装 git，适合后续每日自动更新）

**前置**：安装 Git for Windows（https://git-scm.com/download/win），一路 Next。

装完后在 PowerShell 里：

```powershell
cd c:\Users\tillydu\WorkBuddy\20260421141105\site
git init
git add -A
git commit -m "初版上线：2026-04-22"
git branch -M main
git remote add origin https://github.com/你的用户名/ec-pa-briefing.git
git push -u origin main
```

之后每次更新（比如新增一条信息后）：

```powershell
cd c:\Users\tillydu\WorkBuddy\20260421141105\site
git add -A
git commit -m "update"
git push
```

推上去 1 分钟内 GitHub Pages 会自动重建，同事刷新页面就是最新的。

## 关于每日更新的配合

站点上线后，**每日流程**是这样的：

1. AI 生成今日最新版（在 `C:\Users\tillydu\Desktop\政务信息更新\华东政务信息_YYYY-MM-DD_HHMM\`）
2. 你审核通过后，告诉 AI "同步上线"
3. AI 执行：
   - 复制最新版到 `site/` 覆盖
   - 如果走方式 B，AI 自动跑 `git add / commit / push`（1 分钟内公网可见）
   - 如果走方式 A，AI 生成一个压缩包，你网页重新拖上传

**推荐走方式 B**，一次性装完 git，之后全自动，不用你动手。

## 关于未来是否加密码门

- 现在：不加，任何人拿到链接就能看
- 以后要加：HTML 里加一段 JS 前端密码门（不是真安全，挡一下非相关人员够用），30 秒就能挂上
- 真要做权限管理：需要换部署平台（Cloudflare Pages / Vercel 都支持 Access 控制，免费版够用）

如果同事圈可控，**先不加密码**最省事。

## 链接设计建议

- 仓库名起短点：`ec-pa` / `pa-briefing` / `gov-info` 都行，反正显示在 URL 里
- 如果有自定义域名（比如公司域名），可以在 GitHub Pages 里绑一个 CNAME，链接变成 `https://pa.你的域名.com/`，更专业
